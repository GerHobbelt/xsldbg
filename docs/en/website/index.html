Introduction

xsldbg is a text based tool to debug stylesheets and it is similar to the Unix/Linux debugger gdb. It uses using the libxslt library by http://www.xmlsoft.org/XSLT

  • It is in the pre-beta stage of development. Comments about feature set provided are welcome.
  • Only simple simple XSL commands are supported as yet.: Fixed!. But yet to verify that all commands work as expected

Requirements

   libxml2-2.4.7   or greater

   libxslt-1.0.6     or greater

But I recommend you get the latest available for your distribution. Try

libxslt rpmfind search.

libxml rpmfind search.

A system with  GNU compatible make and build tools

  The "more" system command is available (need by internal help system)

 

Limitations

  • The format of search results needs to be fixed
  • Searching can take a while with large documents ie when debugging docbook.xsl

Installation

Warning!Ensure that libxslt has NOT been configured with  the "--with-debugger=no" option otherwise you will get lots of errors

when xsldbg is compiled.   

 

Extract the latest version of  from http://sourceforge.net/project/showfiles.php?group_id=35673

    make -f  Makefile.cvs

    ./configure

   make

   make install

    

Testing 

     To test installation of xsldbg



         cd xsldbg/docs/en

         xsldbg --shell --root / xsldoc.xsl xsldoc.xml

      And you should get a prompt like
            Starting stylesheet





            Reached template :"/"



            Breakpoint 1 in file xsldoc.xsl : line 21

            (xsldbg) xsl:template >
You then can see the results of continued running through stylesheet by using "cont" command and pressing <ENTER> a few times



                     cont
                     <ENTER>
                     <ENTER>
                     <ENTER>
                     <ENTER>

 The overview section of the help file should have printed out.

 

Help system 

       xsldbg is has built using xml/xslt. To help an overview of commands type
            help
       You can use "help" to get a more detailed information about a command eg
            help help
 

 

Version  notes

 See release notes associated with package. 

These issues are still current

1) The File related commands are disable for the moment.
2) When you do a cat on the style source the output is white space stripped so it will look different to your original source files. This is not a "bug"! This is the default behaviour of the stylesheet processor of libxslt. This tool is intended to be used by other programs which can handle formatting the output.
3) The xsldebugger can be built on windows given correct project files. See TODO's
