Introduction

xsldbg is a text based tool to debug stylesheets it is similar to the Unix/Linux debugger gdb. It uses using the libxslt library by http://www.xmlsoft.org/XSLT
  • It is in the pre-beta stage of development. Comments about feature set provided are welcome.
  • Only simple simple XSL commands are supported as yet.: Fixed
  • Break points to a specific line in the stylesheet are yet made stable : Done!
  • The debugger suffers from crashes "segmentation faults" which is being looked into : Fixed
  • The configure/build scripts need to be redone because they expect KDE to be installed : Done!

Testing

     To test installation of xsldbg

          cd xsldbg/docs/en

         xsldbg --shell --root / xsldoc.xsl xsldoc.xml

      And you should get a prompt like
            Starting stylesheet


            Reached template :"/"

            Breakpoint 1 in file xsldoc.xsl : line 21
            (xsldbg) xsl:template > 

You then can see the results of continued running through stylsheet by using "cont" command and pressing <ENTER> a few times

                     cont
                     <ENTER>
                     <ENTER>
                     <ENTER>
                     <ENTER>

 The overview section of the help file should have printed out.

 

Help system

       xsldbg is has built using xml/xslt. To help an overview of commands type
            help
       You can use "help" to get a more detailed information about a command eg
            help help
 

Command sumamary


A summary of commands you can enter into debugger are 

      Help related : help
     Running related : {bye|exit|quit},step,stepup,stepdown,
                            continue,run
     Template related :templates,where
     Break point related :break,showbreak,delete,enable
     Expression viewing(xpath) :cat
     Node viewing :ls,dir,du,cat,pwd
     Variable viewing :globals,locals,cat
     Node selection :source,data,cd
     File related :validate,load,save,write,free

 
 

    Version  notes

 

 

1) The File related commands are disable for the moment.

2) When you do a cat on the style source the output is white space stripped so it will look different to your original source files. This is not a "bug"! This is the default behaviour of the stylesheet processor of libxslt. This tool is intended to be used by other programs which can handle formatting the output.

3) The xsldebugger can be built on windows given correct project files.
