Introduction

xsldbg is a text based tool to debug stylesheets and it is similar to the Unix/Linux debugger gdb. It uses using the libxslt library by http://www.xmlsoft.org/XSLT

  • It is in the pre-beta stage of development with functionality now frozen until the start of the next beta cycle. Comments about feature set provided are welcome.
  • Only simple simple XSL commands are supported as yet.: Fixed!. But yet to verify that all commands work as expected

Requirements

   libxml2-2.4.10   or greater
   libxslt-1.0.9   or greater
       or
   libxslt-1.0.8 snapshot that has been patched with libxslt-1.0.8_ki_0.8.4.diff from website

Warning!  libxslt may not compile with shared library enabled. I recommend that libxslt be configured using at least these options
    --with-debugger    --enable-shared=no

I recommend you get the latest available for your distribution. Try
    libxslt rpmfind search.
    libxml rpmfind search.

A system with  GNU compatible make and build tools

  The "more" system command is available (need by internal help system)


Limitations

  • The format of search results needs to be fixed
  • Searching can take a while with large documents ie when debugging docbook.xsl
  • Both the stylesheet and xml data file will be reloaded each time xsldbg starts a "run". It is too complex to complex considering keeping track of which files are out of date

Installation

Extract the latest stable version of xsldbg from http://sourceforge.net/project/showfiles.php?group_id=35673 In cvs , for example, the 0.8.4 release is tagged "rel-0-8-4" in cvs. See website for details of the latest stable release.
    make -f  Makefile.cvs
    ./configure --enable-shared=no
   make
   make install


Do a
   ./configure --help
for details of other options to configuring xsldbg


Testing 

     To test installation of xsldbg



         cd xsldbg/docs/en

         xsldbg --shell xsldoc.xsl xsldoc.xml


      And you should get the xsldbg prompt like


            Starting stylesheet


            Reached template :"/"
            Breakpoint 1 in file xsldoc.xsl : line 21
            (xsldbg) xsl:template >

     You then can see the results of continued running through stylesheet by using "cont" command and pressing <ENTER> a few times


                     cont
                     <ENTER>
                     <ENTER>
                     <ENTER>
                     <ENTER>


       The overview section of the help file should have printed out on the screen.

These issues are still current

1) The File related commands are disabled for the moment.
2) When you do a cat on the style source the output is white space stripped so it will look different to your original source files. This is not a "bug"! This is the default behaviour of the stylesheet processor of libxslt. This tool is intended to be used by other programs which can handle formatting the output.

FAQ

Libxslt doesn't compile:

  • Check http://xmlsoft.org/XSLT/ and mail.gnome.org/archives/xslt/ for current issues
  • Ensure that libxslt was configured with
      --with-debugger    --enable-shared=no

xsldbg compiles ok but doesn't run when used from command line:

  • Make sure that you set and exported the correct LD_LIBRARY_PATH
  • Make sure that you've configure the correct libxml and libxslt prefix in libxslt, and xsldbg. Check the values supplied to
       --with-libxslt-prefix
       --with-libxml-prefix

How big is this program: At last count there was over twenty thousand lines of human generated text. Which is about the same size as the libxslt directory in the libxslt distribution.

Why do I get a prompt that looks like:

    (xsldbg) xsldbg_default_node >

    For some reason xsldbg could not load or use the specified stylesheet or xml data file. Try using the "source" or "data" commands and then "run".
