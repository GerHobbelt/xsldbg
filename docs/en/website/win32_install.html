xsldbg Installation for win32 from cvs


xsldbg will ONLY compile when using static libraries there are some exports that still need to be setup
. And you probably will need to change the include and library paths to suit your configuration.
Use batch build to select the static libraries.

First copy the files from win32\dsp\libxml2_new_files and replace those in
  < GNOME-XML >\win32\dsp

Build gnome-xml.

Then copy the files from win32\dsp\libxslt_new_files and replace those in
  < LIBXSLT >\win32\dsp

Build libxslt.

Warning! Make sure that there is a /D "WITH_DEBUGGER" in your compile options for libxslt otherwise xsldbg will
NOT work.

Build xsldbg from < XSLDBG >win32\dsp\xsldbg.dsw.
See index.html for a quick tutorial on xsldbg

It is required that xsldbg, be in your system PATH.An example would be
  set PATH=< XSLDBG >\win32\dsp\xsldbg\%; < GNOME-XML >\win32\dsp\xmlcatalog; %PATH%;



Assuming you checkout xsldbg to C:\Program Files\xsldbg

Set the XSLDBG_DOCS_DIR environment variable to point to xsldbg's documentation. An example
  set XSLDBG_DOCS_DIR="< XSLDBG >\docs\en\"



Setup the XML_CATALOG_FILES environment variable to point to your xml catalog.
An example would be
        set XML_CATALOG_FILES="file:///c:/etc/xml/catalog"


Setup your xml catalogs using xmlcatalog otherwise you will get warnings when running searches
See example xml catalog creation script
